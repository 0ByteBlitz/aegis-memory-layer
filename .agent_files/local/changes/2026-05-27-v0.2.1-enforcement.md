# Plumblines v0.2.1 Enforcement Gate

## What changed

Implemented Plumblines v0.2.1 according to the provided implementation spec.

Created:

- `scripts/plumblines-lib.sh`
- `scripts/check-completeness.sh`
- `docs/record-schema.md`
- `docs/ci-wiring.md`

Replaced:

- `scripts/check-staleness.sh`

Updated:

- `README.md`
- `templates/change.md`
- `templates/project-state.md`
- `templates/working-state.md`
- `templates/domain-state.md`
- `templates/decisions.md`

## Why

v0.2 had two known gaps:

1. The completion check was voluntary and could be skipped silently.
2. Staleness parsing relied on prose-oriented patterns rather than a deterministic metadata contract.

v0.2.1 adds a shared YAML-frontmatter parser, a completeness gate, and CI/hook wiring documentation.

## Validation

Ran acceptance checks in a scratch git repository at `/tmp/plumblines-accept`.

Verified:

1. `chmod +x scripts/*.sh` made all three scripts executable in the scratch repo.
2. `check-staleness.sh` exited clean when no dependency changed after the record commit.
3. `check-staleness.sh` flagged only the related record after its `depends_on` file changed.
4. `check-completeness.sh` flagged a source-touching commit with no change record.
5. `check-completeness.sh` flagged a record with `trust: verified` whose provenance included an `assumed` input.
6. After adding the missing record and downgrading the bad trust label, `check-completeness.sh` exited 0.
7. No runtime dependencies beyond bash, awk, git, find, and coreutils were used. The dependency grep ignored comment lines because the required verbatim library comment mentions `yq/jq`.

## Risks

- The parser assumes well-formed frontmatter.
- Malformed YAML is not fully validated.
- The parser intentionally supports the limited schema shape documented in `docs/record-schema.md`, not arbitrary YAML.
- GitHub file writes through the connector do not expose a chmod mode control, so executable-bit enforcement was validated in the scratch repo with `chmod +x scripts/*.sh`.

## Follow-ups

- Consider a frontmatter-lint step.
- Consider making staleness blocking once the workflow is trusted.
