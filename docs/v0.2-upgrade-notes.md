# Plumblines v0.2 Upgrade Notes

This release upgrades Plumblines from a passive Markdown convention into a stronger project-continuity workflow for AI coding agents.

---

## What changed

### 1. Loading policy

Added `templates/loading-policy.md`.

The loading policy tells agents what to read, what to skip, and how to choose context based on task type.

This helps avoid loading unrelated project history into the context window.

### 2. Commit anchoring

Added `valid_as_of_commit` fields to state templates and change records.

This creates a concrete link between project memory and the code commit it was valid against.

### 3. Staleness checking

Added an optional script:

```txt
scripts/check-staleness.sh
```

The script scans Markdown records under `.agent_files`, reads their `valid_as_of_commit`, and flags records whose listed dependency files changed after that commit.

### 4. Provenance tracking

Added `templates/provenance.md`.

Provenance records what the agent relied on, whether assumptions were used, and how much confidence the decision deserves.

### 5. Completion checks

Updated the change template to include a completion check.

A task that changes code or durable project documentation should not be considered complete until the change record, validation, risks, and follow-ups are recorded.

---

## Why this matters

The original version of Plumblines provided good structure, but relied heavily on agent compliance.

v0.2 adds stronger mechanics:

- clearer context selection
- commit-based validity
- optional stale-record detection
- provenance tracking
- explicit completion checks

The framework remains plain Markdown and zero-dependency by default. The staleness checker is optional and only requires Git plus a POSIX shell.

---

## Recommended migration

1. Add `valid_as_of_commit` to existing state files.
2. Add `Files This State Depends On` sections to state files.
3. Update change records to include `Validity` and `Touched Files`.
4. Copy `templates/loading-policy.md` into `.agent_files/LOADING_POLICY.md`.
5. Run `scripts/check-staleness.sh` occasionally before agent work or before merging a branch.
